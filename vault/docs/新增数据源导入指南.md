# 新增数据源导入指南

本指南说明如何基于 `BaseIngestor` 抽象基类创建新的数据源导入类，以便将不同格式的数据导入到 Vault 存储系统中。

## 概述

数据导入流程使用了模板方法模式，通过继承 `BaseIngestor` 抽象基类来实现具体的数据源处理逻辑。每个数据源类需要实现两个核心方法：`prepare_tasks()` 和 `process_tasks()`。

## BaseIngestor 抽象基类

### 核心方法

- **`prepare_tasks()`**: 准备所有需要处理的任务单元
- **`process_tasks()`**: 处理一小批任务，生成 PackSequence 对象
- **`run()`**: 执行完整的数据导入流程（已实现，无需重写）

### 配置参数

```python
@dataclass
class YourIngestor(BaseIngestor):

    # ... 其他参数
```

## 实现步骤

### 1. 创建数据类并继承 BaseIngestor

```python
from dataclasses import dataclass
from typing import Any, Generator
import vault.schema.multimodal as multimodal

@dataclass
class YourDataIngestor(BaseIngestor):
    name: str = "your-data-source"
    vault_path: str = "/path/to/vault/storage"
    
    task_chunk_size: int = 100
    num_workers: int = 8
    
    # 自定义配置
    data_root: str = "/path/to/your/data"
    metadata_file: str = "/path/to/metadata.jsonl"
```

### 2. 实现 prepare_tasks() 方法

此方法负责扫描数据源，准备所有需要处理的任务单元。任务单元可以是文件路径、数据组、或任何便于处理的数据结构。

```python
def prepare_tasks(self) -> list[Any]:
    """
    准备所有需要处理的任务单元。
    
    Returns:
        list[Any]: 任务列表，每个元素代表一个处理单元
    """
    tasks = []
    
    # 示例：扫描文件或读取元数据
    # 方式1：扫描文件系统
    for file_path in scan_data_files(self.data_root):
        tasks.append(file_path)
    
    # 方式2：从元数据文件读取
    # with open(self.metadata_file) as f:
    #     for line in f:
    #         data = json.loads(line)
    #         tasks.append(data)
    
    return tasks
```

### 3. 实现 process_tasks() 方法

此方法在并发的工作进程中执行，负责处理具体的数据转换逻辑。

```python
def process_tasks(
    self, tasks_chunk: list[Any], worker_id: int
) -> Generator[multimodal.PackSequence, None, None]:
    """
    处理一批任务，生成 PackSequence 对象。
    
    Args:
        tasks_chunk: 分配给此工作进程的任务列表
        worker_id: 工作进程ID，可用于进度条定位
        
    Yields:
        multimodal.PackSequence: 处理完成的数据包
    """
    from tqdm import tqdm
    
    for task in tqdm(
        tasks_chunk, 
        desc=f"Worker-{worker_id}", 
        position=worker_id % 8 + 1
    ):
        # 处理单个任务
        # 1. 加载数据（图像、文本等）
        image_data = load_image(task["image_path"])
        text_content = task["caption"]
        
        # 2. 创建多模态对象
        image = multimodal.Image.create(
            image_data,
            uri=task["image_uri"],
            source=f"{self.name}/{task['subset']}",
            annotations=[],  # 添加相关注释
        )
        
        text = multimodal.Text.create(
            content=text_content,
            uri=task["text_uri"],
            source=f"{self.name}/{task['subset']}",
            language="zh",  # 或其他语言
        )
        
        # 3. 生成 PackSequence
        yield multimodal.PackSequence.from_text_to_image(
            caption=text,
            image=image,
            source=f"{self.name}/{task['subset']}",
            uri=task["uri"],
            meta=task.get("metadata", {}),
        )
```

## 多模态对象创建

### 图像对象

```python
image = multimodal.Image.create(
    image_data,              # 图像二进制数据
    uri="/path/to/image",    # 唯一标识符
    source="dataset/subset", # 数据来源
    annotations=[           # 注释列表
        multimodal.Annotation.generated_by(model="model_name"),
        multimodal.Annotation.create(name="tag", type_="classification")
    ],
)
```

### 文本对象

```python
text = multimodal.Text.create(
    content="文本内容",
    uri="/path/to/text",
    source="dataset/subset",
    language="zh",  # 语言标识
    annotations=[],
)
```

### PackSequence 创建

根据数据类型选择合适的创建方法：

```python
# 文本到图像任务
PackSequence.from_text_to_image(caption=text, image=image, ...)

# 奖励模型数据
PackSequence.from_t2i_reward(caption=text, image=images, ...)

# 通用方法
PackSequence.create(texts=[text], images=[image], ...)
```

## 最佳实践

### 1. 错误处理
```python
try:
    # 数据处理逻辑
    image = load_image(path)
except Exception as e:
    logger.warning(f"处理 {path} 时出错: {e}")
    continue  # 跳过错误数据
```

### 2. 内存管理
- 避免在内存中加载过多数据
- 及时释放不需要的对象
- 考虑使用生成器模式

### 3. 进度监控
```python
for item in tqdm(items, position=worker_id % 8 + 1, desc=f"Worker-{worker_id}"):
    # 处理逻辑
```

### 4. 元数据管理
- 保持元数据结构一致
- 添加有意义的源标识和URI
- 合理使用注释系统

## 调试技巧

1. **小批量测试**: 先用少量数据测试逻辑正确性
2. **单进程调试**: 设置 `num_workers=1` 便于调试
3. **日志输出**: 合理使用 logger 记录关键信息
4. **数据验证**: 检查生成的 PackSequence 对象是否符合预期
